# Global Gemini CLI Preferences

## Communication
- Be concise. Skip preambles.
- When showing code changes, show the full file with the change in context.

## Code Style
- Prefer TypeScript over JavaScript when the project supports it.
- Use async/await — never raw callbacks.
- Write clear variable names. No single-letter names outside loop counters.

## Workflow
- After making code changes, always run the project's test suite.
- When creating new files, check for an existing pattern in the project first.
